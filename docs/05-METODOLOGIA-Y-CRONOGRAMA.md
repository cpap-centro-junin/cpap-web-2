# Metodología y Cronograma

## Metodología de Trabajo

El proyecto se desarrolla siguiendo una **metodología incremental organizada por fases**, que permite un adecuado control del avance y validación de cada etapa.

---

## Fases del Proyecto

### Fase 1: Análisis y Recolección de Requerimientos
**Objetivo:** Entender las necesidades y definir la estructura del sistema

**Actividades:**
- Análisis del sitio web actual
- Recopilación de información sobre requerimientos funcionales y no funcionales
- Definición de la estructura general del sitio
- Identificación de necesidades específicas de la institución

**Entregable:** Documento de requerimientos y especificaciones

---

### Fase 2: Diseño del Sistema
**Objetivo:** Crear la arquitectura y diseño visual del sitio

**Actividades:**
- Elaboración del diseño de arquitectura del sistema web
- Definición de estructura de secciones
- Diseño de navegación del sitio
- Organización de la información
- Establecimiento de lineamientos de diseño visual
- Definición de identidad gráfica del sitio web

**Entregable:** Diseños, mockups y arquitectura del sistema

---

### Fase 3: Desarrollo del Sistema
**Objetivo:** Implementar el frontend y backend

**Actividades:**
- Desarrollo del frontend utilizando HTML5, CSS3 y JavaScript
- Desarrollo del backend con Laravel
- Implementación de funcionalidades definidas
- Integración con la base de datos MySQL
- Implementación de sistema de administración

**Entregable:** Sistema web funcional

---

### Fase 4: Pruebas y Validación
**Objetivo:** Verificar el correcto funcionamiento del sistema

**Actividades:**
- Pruebas funcionales de todas las secciones
- Pruebas de rendimiento
- Verificación de compatibilidad con diferentes navegadores
- Pruebas de responsive design
- Corrección de errores detectados
- Validación con los requerimientos establecidos

**Entregable:** Sistema validado y optimizado

---

### Fase 5: Implementación y Publicación
**Objetivo:** Poner el sistema en producción

**Actividades:**
- Despliegue en servidor de hosting
- Configuración del dominio
- Instalación del certificado SSL
- Migración de datos (si aplica)
- Pruebas finales en entorno de producción
- Capacitación básica al personal administrativo

**Entregable:** Sistema web operativo en internet

## Gestión del Proyecto

### Herramientas de Gestión
- **GitHub Projects:** Para seguimiento de tareas
- **Issues:** Para gestión de bugs y mejoras
- **Pull Requests:** Para revisión de código
- **Branches:** Para desarrollo organizado

### Reuniones de Coordinación
- **Semanales:** Revisión de avances con el equipo
- **Quincenales:** Presentación de avances a la institución
- **Ad-hoc:** Según necesidades de validación o aprobación

### Control de Calidad
- Revisión de código entre pares
- Pruebas continuas durante el desarrollo
- Validación de requerimientos en cada fase
- Documentación del código

---

## Riesgos y Mitigación

| Riesgo | Probabilidad | Impacto | Mitigación |
|--------|--------------|---------|------------|
| Cambios en requerimientos | Media | Alto | Documentación clara desde el inicio |
| Problemas técnicos de hosting | Baja | Medio | Selección de proveedor confiable |
| Retrasos en desarrollo | Media | Medio | Buffer de tiempo en el cronograma |
| Incompatibilidad de navegadores | Baja | Bajo | Pruebas continuas en múltiples navegadores |
